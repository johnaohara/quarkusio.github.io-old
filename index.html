Quarkus

is a Cloud-Native, Container-First stack
for creating Java applications tailored for Graal VM & HotSpot, crafted from the best of breed Java libraries and standards.

Get Started with Quarkus

Imperative and Reactive in one programming model

Use the familiar imperative code approach or need the event-driven and non-blocking approach. Quarkus brings both under one roof. Brings serverless to Java.

Container First

Quarkus is made for optimally running on Graal VM and HotSpot. Low boot time, low RSS memory (not just heap size!) offering stellar boot and memory utilization in containers and container orchestration platforms like Kubernetes. We use a technique we call compile time boot.

Developer Joy

Cohesive platform from config to build.
Streamlined code for the 80% common usages, flexible for the 20%. Instant fix the bug and hit reload development loop for all libraries including Hibernate.

Best of Breed Libraries and Standards
